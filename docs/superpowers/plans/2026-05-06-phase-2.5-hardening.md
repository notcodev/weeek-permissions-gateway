# Phase 2.5 — Pre–Phase-3 Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Address three hardening items the Phase 2 final review surfaced — keyed-HMAC fingerprint instead of plain SHA-256 (offline-dictionary resistance for the workspace uniqueness probe), constraint-aware unique-violation detection (kill the brittle `.cause` walk), and explicit Origin-header CSRF defense on the tRPC mutation surface (defense-in-depth on top of `SameSite=Lax`).

**Architecture:** Three small, independent diffs. (1) `crypto/fingerprint.ts` swaps `createHash("sha256")` for `createHmac("sha256", pepper)`, sourcing a new `FINGERPRINT_HMAC_PEPPER` env var; the existing `weeek_workspace_owner_fingerprint_uq` index keeps the same column type and semantics — fingerprints are recomputed on import, no row migration needed because no prod data exists. (2) `routers/workspace.ts` `isUniqueViolation` becomes a constraint-name check (`weeek_workspace_owner_fingerprint_uq`) walking the `.cause` chain so a future schema with multiple unique indices can't conflate violations. (3) A pure `isOriginAllowed(req, allowed)` helper lands in `src/server/trpc/origin-check.ts` and is invoked from `src/app/api/trpc/[trpc]/route.ts` before `fetchRequestHandler`; mutations from a browser with a mismatched `Origin` get a 403 before any procedure runs.

**Tech Stack:** Same as Phase 2 — Node 22, Next.js 16, tRPC v11, TypeScript 6 strict, Drizzle ORM, vitest with msw + testcontainers, pnpm 9. No new runtime deps.

**Spec reference:** `docs/superpowers/specs/2026-05-06-weeek-permissions-gateway-design.md` §8 ("CSRF: Better Auth token + Origin check on tRPC mutations") and §12 ("Sub-key storage: HMAC-SHA256(rawKey, SUB_KEY_HMAC_PEPPER)" — same pepper-keyed-HMAC pattern, applied here to the workspace fingerprint).

**Out of scope for this phase:** Master-key encryption rotation infrastructure (`encVersion>1`, multi-active-key store, re-encrypt path) — explicitly deferred per the controller's brief. Sub-key issuance lands in Phase 3 with its own `SUB_KEY_HMAC_PEPPER`. No new policy enforcement.

### Task 1 — HMAC-keyed fingerprint + `FINGERPRINT_HMAC_PEPPER`

**Files:**
- Modify: `src/server/crypto/fingerprint.ts`
- Modify: `tests/unit/fingerprint.test.ts`
- Modify: `.env.example`
- Modify: `tests/setup.ts`
- Modify: `tests/integration/workspace-router.test.ts`

The fingerprint is the workspace-uniqueness probe: it lets us reject re-import of the same Weeek master token under the same owner (`UNIQUE (owner_type, owner_id, master_key_fingerprint)`). With plain SHA-256, anyone with read access to `weeek_workspace.master_key_fingerprint` can confirm a guessed master token is present. Switching to HMAC-SHA256 with a server-only pepper makes the probe useless without DB *and* env-var compromise.

The pepper is a separate env var from `MASTER_KEY_ENC_KEY` (different blast radius — leaking the pepper does not expose master keys; leaking the encryption key does not let the attacker probe for a known token). Both are 32 random bytes, base64-encoded — `pnpm genkey` produces them; same script, run twice.

- [x] **Step 4: Replace `src/server/crypto/fingerprint.ts`**

```ts
import { createHmac } from "node:crypto";

const PEPPER_BYTES = 32;

function getPepper(): Buffer {
  const b64 = process.env.FINGERPRINT_HMAC_PEPPER;
  if (!b64) throw new Error("FINGERPRINT_HMAC_PEPPER is required");
  const raw = Buffer.from(b64, "base64");
  if (raw.byteLength !== PEPPER_BYTES) {
    throw new Error(
      `FINGERPRINT_HMAC_PEPPER must decode to ${PEPPER_BYTES} bytes (got ${raw.byteLength})`,
    );
  }
  return raw;
}

export function fingerprint(rawKey: string): Uint8Array {
  return new Uint8Array(createHmac("sha256", getPepper()).update(rawKey, "utf8").digest());
}

export function last4(rawKey: string): string {
  return rawKey.length <= 4 ? rawKey : rawKey.slice(-4);
}
```

The pepper is read on every call (not cached) — that's intentional. The cost is negligible and it keeps the test surface honest.

### Task 2 — Constraint-aware `isUniqueViolation`

**Files:**
- Modify: `src/server/trpc/routers/workspace.ts`

The Phase 2 helper walks `err.cause` up to 5 levels looking for SQLSTATE `23505`. That's a heuristic — it'll return `CONFLICT` for *any* unique-violation. Drizzle 0.45 wraps postgres-js errors in `DrizzleQueryError` with the original `PostgresError` on `.cause`. `PostgresError` carries both `code` (SQLSTATE) and `constraint_name`. Check both.

- [x] **Step 1: Update `isUniqueViolation` and the import call site in `src/server/trpc/routers/workspace.ts`**

Replace the existing helper with:

```ts
const FINGERPRINT_CONSTRAINT = "weeek_workspace_owner_fingerprint_uq";

function isUniqueViolationOnConstraint(err: unknown, constraint: string): boolean {
  let cur: unknown = err;
  for (let i = 0; i < 5 && cur; i++) {
    if (
      typeof cur === "object" &&
      cur !== null &&
      (cur as { code?: string }).code === "23505" &&
      (cur as { constraint_name?: string }).constraint_name === constraint
    ) {
      return true;
    }
    cur = (cur as { cause?: unknown }).cause;
  }
  return false;
}
```

Update the catch site:

```ts
} catch (err) {

    throw new TRPCError({
      code: "CONFLICT",

    });
  }
  throw err;
}
```

### Task 3 — tRPC Origin check on mutations

**Files:**
- Create: `src/server/trpc/origin-check.ts`
- Create: `tests/unit/origin-check.test.ts`
- Modify: `src/app/api/trpc/[trpc]/route.ts`

`SameSite=Lax` cookies (Phase 1) already block CSRF for cross-origin POSTs. Spec §8 mandates an explicit Origin check too — defense in depth.

Policy:
- `GET` / `HEAD` / `OPTIONS` → always allowed
- Mutations with `Origin` matching allowed → allowed
- Mutations with present-but-mismatched `Origin` → blocked (403 `{"error":"origin_mismatch"}`)
- Mutations with no `Origin` header → allowed (server-to-server)

Allowed origin from `BETTER_AUTH_URL`. Trailing slashes stripped on both sides.

- [x] **Step 3: Write `src/server/trpc/origin-check.ts`**

```ts
const SAFE_METHODS = new Set(["GET", "HEAD", "OPTIONS"]);

function normalise(origin: string): string {
  return origin.replace(/\/+$/, "");
}

export function isOriginAllowed(req: Request, allowed: string): boolean {
  if (SAFE_METHODS.has(req.method.toUpperCase())) return true;
  const origin = req.headers.get("origin");
  if (origin === null) return true;
  return normalise(origin) === normalise(allowed);
}
```

```ts
import { fetchRequestHandler } from "@trpc/server/adapters/fetch";
import { createTRPCContext } from "@/server/trpc/init";
import { appRouter } from "@/server/trpc/routers";
import { isOriginAllowed } from "@/server/trpc/origin-check";

function getAllowedOrigin(): string {
  return process.env.BETTER_AUTH_URL ?? "http://localhost:3000";
}

const handler = (req: Request) => {
  if (!isOriginAllowed(req, getAllowedOrigin())) {
    return new Response(JSON.stringify({ error: "origin_mismatch" }), {
      status: 403,
      headers: { "content-type": "application/json" },
    });
  }
  return fetchRequestHandler({
    endpoint: "/api/trpc",
    req,
    router: appRouter,
    createContext: (opts) => createTRPCContext(opts),
  });
};

export { handler as GET, handler as POST };
```
