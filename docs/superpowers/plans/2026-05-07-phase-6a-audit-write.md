# Phase 6a — Audit Log Write Path

**Goal:** Capture every proxy request that has a workspace context (post-auth) into a per-tenant `audit_log` table. Reader UI and retention cron land in 6b/6c.

## Schema (`audit_log`)

| Column | Type | Notes |
|--------|------|-------|
| `id` | text PK (cuid) | |
| `workspace_id` | text NOT NULL FK→`weeek_workspace` | `ON DELETE CASCADE` — workspace removal wipes audit |
| `sub_key_id` | text NULLABLE FK→`sub_key` | `ON DELETE SET NULL` — sub-key revoke preserves history |
| `request_id` | text NOT NULL | UUID generated in handler |
| `method` | text NOT NULL | uppercased verb |
| `path` | text NOT NULL | proxied path with no `/api/v1` prefix |
| `query` | text NULLABLE | full query with leading `?`, truncated to 500 chars |
| `our_status` | int NOT NULL | gateway response status |
| `upstream_status` | text NOT NULL | `"200"` / `"network_error"` / `"timeout"` / `"n/a"` (deny) |
| `latency_ms` | int NOT NULL | wall clock from handler entry to response |
| `verb` | text NULLABLE | matched verb; null on `unknown_route` and pre-auth fail |
| `deny_reason` | text NULLABLE | `unknown_route` / `verb_missing` / `project_not_in_scope` / `board_not_in_scope`; null on success |
| `ip_hash` | bytea NULLABLE | HMAC-SHA256 of first hop in `x-forwarded-for`, peppered with `FINGERPRINT_HMAC_PEPPER` |
| `user_agent` | text NULLABLE | UA header truncated to 200 chars |
| `created_at` | timestamptz NOT NULL DEFAULT now() | |

**Indexes:** `(workspace_id, created_at)`, `(sub_key_id, created_at)`, `(workspace_id, deny_reason)`.

Migration: `0003_nice_raider.sql` (auto-generated by `pnpm db:generate`).

## Write path

`src/server/proxy/audit.ts` gains `recordAudit(input)` alongside the existing `recordUsage(subKeyId)`. Both are best-effort, fire-and-forget, errors swallowed via `logger.warn`. Pure helpers `hashClientIp`, `truncateUserAgent`, `truncateQuery` are exported for unit testing.

`handler.ts` captures `Date.now()` and request headers (`x-forwarded-for`, `user-agent`) at entry. After every response that has a workspace (post-auth), `void recordAudit(...)` fires before returning. Pre-auth 401s skip audit (no workspace FK).

Audit covers four code paths:
- `unknown_route` → deny_reason recorded, verb null, upstreamStatus `"n/a"`
- policy deny → deny_reason recorded, verb recorded, upstreamStatus `"n/a"`
- success → deny_reason null, upstreamStatus `"200"` (or whatever Weeek returned)
- upstream 502 → deny_reason null, upstreamStatus `"network_error"`
